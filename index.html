在类中封装属性名

在面向对象编程中，封装属性名的目的是隐藏类的内部实现细节，通过受控的方法访问或修改数据，从而提高代码的安全性和可维护性

一、 Python中的属性封装实现

  1. 使用私有属性（名称改写）
    * 在属性名前添加双下划线__, Python会将其名称改写为 `_ClassName__attribute`，限制直接访问。
* 提供公有方法（getter/setter）或 `@property` 装饰器来访问和验证数据。

class Person:
    def __init__(self, name, age):
        self.__name = name  # 私有属性
        self.__age = age

    # 使用@property 访问属性
    @property
    def name(self):
        return self.__name

    # 定义setter方法验证数据
    # 用于控制属性赋值逻辑的 setter 方法，它会验证你给属性赋的值是否合法
    @name.setter
    def name(self, value):
        if isinstance(value, str) and len(value) > 0:
            self.__name = value
        else:
            raise ValueError("Invalid name")

    @property
    def age(self):
        return self.__age

    @age.setter
    def age(self, value):
        if value >= 0:
            self.__age = value
        else:
            raise ValueError("Age cannot be negative")

# 使用示例
p = Person("Alice", 30)
print(p.name)  # 输出: Alice
p.age = 25     # 合法操作
p.age = -5     # 抛出 ValueError

p.name = "Bob"
p.age = 44
print(p._Person__name)
print(p._Person__age)
  1. @name.setter 的作用

  • 功能：这是一个装饰器，用来标记这是一个 setter 方法。当你给 name 属性赋值时（例如 p.name = "Bob"），自动触发这个方法。

  • 为什么需要：普通的属性赋值（如 p.name = 123）无法验证数据是否合法，而 setter 方法允许你在赋值前插入自定义检查逻辑。

  1. 方法内部的验证逻辑
def name(self, value):
    if isinstance(value, str) and len(value) > 0:  # 检查是否为非空字符串
        self.__name = value  # 合法则赋值给私有属性 __name
    else:
        raise ValueError("Invalid name")  # 非法则抛出异常

    * **`isinstance(value, str)`** ：检查 `value` 是否是字符串类型（例如防止数字或对象被赋值给 `name`）。
    * **`len(value) > 0`** ：确保字符串不是空的（例如防止 `name = ""`）。
  • raise ValueError ：如果值不合法，直接抛出错误，阻止赋值。
  1. 和 @property 的关系
  • @property 定义 getter：负责读取属性时的逻辑（例如 print(p.name)）。
  • @name.setter 定义 setter：负责写入属性时的逻辑（例如 p.name = "Bob"）。
  • 配套使用：两者共同实现对属性的安全读写控制。
  1. 为什么用 self.__name
  • 双下划线 __name ：这是 Python 的私有属性命名约定，实际会被改写为 _Person__name，防止外部直接访问（如 p.__name 会报错，但 p._Person__name 仍可访问，不过约定上不这样做）。
  • 封装目的：强制通过 setter 方法修改属性，确保所有赋值都经过验证。

二、封装的优势

  1. 数据保护：防止外部代码直接修改属性，避免非法值（如负数的年龄）。
  2. 灵活性：可在方法中添加额外逻辑（如数据验证、日志记录）。
  3. 兼容性：内部实现变化时，不影响外部代码（如重命名属性）。

三、单下划线和双下划线的区别

一、单下划线 _name : 受保护的属性（Portected）
1. 命名的约定：

    *  _name 表示这是一个内部属性，暗示开发者“不要直接访问，除非你知道自己在做什么”。
    *  不触发名称改写：名称保持为 _name, 没有自动重命名。
    *  访问权限：外部代码仍可以直接读写（p.name = 123）,但约定上应通过方法或属性装饰器操作。
  1. 典型用途：
  • 类的内部状态管理（如缓存、临时变量）
  • 子类继承时可以安全覆盖或者扩展
class Person:
    def __init__(self, name):
        self._name = name  # 单下划线，约定上的“受保护”属性

p = Person("Alice")
print(p._name)  # 直接访问：输出 "Alice"（不推荐但允许）
二、对比表格
特性 单下划线 _name 双下划线 __name
名称改写 无 有（改写为 _ClassName__name）
直接外部访问 允许（不推荐） 不允许（需改写后的名称）
继承冲突防护 无，子类可直接覆盖 有，避免子类同名属性冲突
用途 约定上的内部属性 强制私有属性
三、何时选择哪种？
  1. 用单下划线 _name ：
    • 属性需要被子类继承或覆盖。
    • 仅作为内部状态管理，不涉及敏感数据。
    • 团队约定明确时（如 Django 框架常用 _ 表示内部方法）。
  2. 用双下划线 __name ：
    • 明确禁止外部直接访问（即使通过名称改写也不建议）。
    • 避免多级继承中的属性命名冲突。
    • 需要强制封装时（如属性值必须通过方法验证）。
四、代码示例
class Base:
    def __init__(self):
        self._protected = "Base Protected"  # 单下划线
        self.__private = "Base Private"     # 双下划线

class SubClass(Base):
    def __init__(self):
        super().__init__()
        self._protected = "Sub Protected"   # 直接覆盖父类的 _protected
        self.__private = "Sub Private"      # 实际名称是 _SubClass__private

    def show(self):
        print(self._protected)              # 输出 "Sub Protected"
        print(self.__private)               # 输出 "Sub Private"
        print(self._Base__private)           # 输出 "Base Private"（访问父类私有属性）

obj = SubClass()
obj.show()
五、总结
  • 单下划线 _name ：约定上的“内部属性”，轻量级封装。
  • 双下划线 __name ：强制私有属性，通过名称改写避免冲突。
  • 实际开发中，优先用单下划线，除非明确需要避免继承冲突或强制封装。

对象支持上下文管理协议

​ 在 Python 中，让对象支持上下文管理协议，简单说就是让你的对象能和 with 语句配合使用，自动处理资源的打开和关闭（比如文件、网络连接、锁等），避免资源泄漏。

只需要在类中实现两个方法：

  1. enter(self)

    • 当with语句开始调用时，返回你需要管理的资源（比如打开的文件对象）。

    • 通常返回self, 但也可以返回其他对象。

  2. exit(self，exc_type，exc_val，exc_tb)

    • 当 with 语句块结束时（无论是否发生异常）调用。

    • 负责清理工作（比如关闭文件、释放锁）。

# 模拟文件打开器
class FileOpener:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode

    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file  # 返回文件对象，供 with 块内使用

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.file.close()  # 确保文件被关闭

# 使用 with 语句自动管理文件
with FileOpener("test.txt", "w") as f:
    f.write("Hello, Context Manager!")
# 这里文件已自动关闭，无需手动调用 f.close()

✅ 为什么要用上下文管理？

  1. 资源安全：即使代码块中发生异常，__exit__ 也会被调用，确保资源释放。
  2. 代码简洁：避免了冗长的 try...finally 结构。
  3. 可读性高：明确展示资源的作用域。

自定义字符串的格式化 

_formats = {
    'ymd' : '{d.year}-{d.month}-{d.day}',
    'mdy' : '{d.month}/{d.day}/{d.year}',
    'dmy' : '{d.day}/{d.month}/{d.year}'
    }

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    def __format__(self, code):
        if code == '':
            code = 'ymd'
        fmt = _formats[code]
        return fmt.format(d=self) # 替换为 self.year, self.month, self.day 的实际值
ddd = Date(2012, 12, 21)
print(format(ddd, 'ymd'))
print('The date is {:mdy}'.format(ddd))


import math
# __format__ 专门用于格式化输出，支持动态格式控制（如精度、单位等）
# __str__ 和 __repr__ 用于一般字符串表示。
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __format__(self, format_spec):
        if format_spec == 'r':  # 直角坐标
            return f"Vector({self.x}, {self.y})"
        elif format_spec == 'p':  # 极坐标
            r = math.hypot(self.x, self.y)
            theta = math.degrees(math.atan2(self.y, self.x))
            return f"Vector(r={r:.2f}, θ={theta:.2f}°)"
        elif format_spec == '':  # 默认格式
            return f"Vector({self.x}, {self.y})"
        else:
            raise ValueError(f"Unknown format specifier '{format_spec}'")

# 使用示例
v = Vector(3, 4)

print(f"{v}")          # 输出: Vector(3, 4)
print(f"{v:r}")        # 输出: Vector(3, 4)
print(f"{v:p}")        # 输出: Vector(r=5.00, θ=53.13°)
print(format(v, 'p'))  # 输出: Vector(r=5.00, θ=53.13°)

linux命令lsof

1.lsof简介

lsof(list open files)可以列出当前系统中进程打开的所有文件，在Linux环境下，我们可以理解为一切(包括网络套接口)皆文件。在实际使用过程中，lsof是一款非常强大的系统监控和系统诊断工具。在终端下输入lsof 即可显示系统打开的文件， lsof 一般需要访问核心内存和各种文件，所以必须以 root 用户的身份运行它才能够充分地发挥其功能。

改变对象字符串显示

1. repr函数简介

​ 在Python中，__repr__和__str__都是用于对象字符串表示的特殊方法，但用途不同。__repr__（representation）旨在提供对象的“官方”字符串表示，通常用于调试和日志，应尽量明确且无歧义（如eval(repr(obj))能重建对象），开发者可通过repr(obj)或直接输出对象时调用；__str__（string）则面向用户，提供更友好、可读性高的描述（如print(obj)时触发），若未定义则默认回退到__repr__。两者均需返回字符串，但__repr__更强调准确性，__str__侧重易读性。

class Pair:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return 'Pair({0.x!r}, {0.y!r})'.format(self)

    def __str__(self):
        return '({0.x!s}, {0.y!s})'.format(self)
p = Pair(1,2)
print(repr(p)) # Pair(1, 2)  # 当调用内置函数repr(obj)时，实际上会调用obj.__repr__()方法
print(str(p)) # (1, 2)
new_p = eval(repr(p))
print("new_p: ", new_p) # new_p:  (1, 2)


p = Pair(True, [1, 2, 3])

print(repr(p))  # 输出：Pair(True, [1, 2, 3])
print(str(p))   # 输出：(True, [1, 2, 3])
# 可以看到 !r 会保留数据的完整类型信息，而 !s 会直接输出值的字符串形式


# 实例：
class Vector3D:
    def __init__(self, x, y, z):

        self.z = z

        # return f"Vector3D({self.x!r},{self.y!r},{self.z!r})"
        return f"{self.__class__.__name__}({self.x!r},{self.y!r},{self.z!r})"
    
    #定制化 str 格式
    def __str__(self):
        return f"3D向量 → X:{self.x:.2f}, Y:{self.y:.2f}, Z:{self.z:.2f}"
print('---------------------------------分割线-----------------------------------')

print(repr(V))
new_v = eval(repr(V))
print(new_v)

2. 案例示范

"""
题目：智能坐标点类
请实现一个 SmartPoint 类，要求满足以下所有条件：
1. 基础要求
包含 x, y, tag 三个属性（tag 为字符串类型的标签）
__repr__ 返回的字符串必须能够通过 eval() 重建对象
__str__ 返回人类可读的格式："标签：(x, y)"（坐标保留2位小数）

2. 特殊需求
当 tag 包含空格时，__repr__ 必须自动添加引号
如果 tag 为空字符串，__str__ 显示 "未命名点"
在列表中打印多个 SmartPoint 对象时，显示的是 __repr__ 格式
"""
class SmartPoint:
    def __init__(self, x, y, tag=''):
        self.x = x
        self.y = y
        self.tag = tag
    def __repr__(self):
        return f"{self.__class__.__name__}({self.x!r}, {self.y!r}, {self.tag!r})"
    
    def __str__(self):
        # if self.tag == "":  # 如果 tag 为空字符串
        if not self.tag:
            return f"未命名点：({self.x:.2f}, {self.y:.2f})"  # 返回默认提示
        else:
            return f"{self.tag}: ({self.x:.2f}, {self.y:.2f})"

# 创建对象
p1 = SmartPoint(3.1415926, 2.71828, "Alpha")
p2 = SmartPoint(-5, 4.2, "")
p3 = SmartPoint(0, 0, "Beta Point")

# 测试1：直接打印
print(p1)           # 应输出：Alpha：(3.14, 2.72)
print(p2)           # 应输出：未命名点：(-5.00, 4.20)

# # 测试2：repr相关

接口测试学习笔记

1. HTTP 与 HTTPS 的概念

  • HTTP
    • 超文本传输协议
    • 是一个基于请求与响应，无状态的，应用层的协议
    • 常基于 TCP/IP 协议传输数据
    • 以明文方式发送内容
  • HTTPS
    • 安全套接字层超文本传输协议
    • 通过计算机网络进行安全通信的传输协议
    • 在 HTTP 的基础上加入了 SSL/TLS 协议
    • HTTP + 加密 + 认证 + 完整性保护 = HTTPS

2. HTTP 与 HTTPS 的区别

  1. HTTPS 比 HTTP 更安全

    • HTTP 是超文本传输协议，连接简单无状态，信息明文传输

    • HTTPS 通过 SSL/TLS 提供安全方式

  2. HTTP 和 HTTPS 使用完全不同的连接方式

  3. HTTP 和 HTTPS 用的端口也不一样

    • HTTP 是 80 端口

    • HTTPS 是 443 端口

  4. HTTPS 协议需要到 CA 申请证书，可能需要一定费用

3.get、post区别

  • HTTP 的 method 字段不同

  • POST 可以附加 body，可以支持 form、json、xml、binary 等各种数据格式

  • 行业通用的规范

    • 无状态变化的建议使用 GET 请求
    • 数据的写入与状态修改建议用 POST
  • 传送数据量不同
  • 安全性不同

4 session、cookie、token的区别

  • Session：数据存储到服务端，只把关联数据的一个加密串放到 Cookie 中标记
  • Cookie：浏览器接受服务器的 Set-Cookie 指令，并把 cookie 保存到电脑上，每个网站保存的 cookie 只作用于自己的网站
  • Token：是一个用户请求时附带的请求字段，用于验证身份与权限
4.1 session的实现机制

​ 服务器创建 Session 出来后，会把 Session id，以 Cookie 的形式回写给客户端 浏览器，这样，只要客户端的浏览器不关，再去访问服务器时，都会带着 Session id ，服务器发现客户机浏览器带 Session id 过来了，就会使用内存中与之对应的 Session 为之服务。(Session 会存在服务器)

4.2 cookie 实现机制
  • Cookie 通过在客户端记录信息确定用户身份，
  • Cookie 由服务器生成，发送给浏览器，浏览器把 Cookie 以 kv 形式保存到某个目录下的文本文件内，下一次请求同一网站时会把该 Cookie 发送给服务器。

4.3 Token 的原理

  • 基于 Token 的身份验证是无状态的，我们不将用户信息存在服务器或 Session 中。这解决了在服务端存储信息时的许多问题 NoSession 意味着你的程序可以根据需要去增减机器，而不用去担心用户是否登录。

  • 基于 Token 的身份验证的过程如下:

    • 用户发送数据请求
    • 程序验证
    • 程序返回一个签名的 token 给客户端
    • 客户端储存 token，并且每次用于每次发送请求。
    • 服务端验证 token 并返回数据。

​ token 通常在 HTTP 的头部发送给服务端

1. 用户参数组件介绍

​ 用户参数（User Parameter）组件在前置处理器元件中添加，前置处理器的作用是在取样器发出请求之前，执行一些操作。即：如果将前置处理器附加到取样器元件中，则它将在该取样器元件运行之前执行。常用操作如：取样器参数设置、环境变量设置、脚本预处理等。之前我们介绍过，JMeter中常用的参数化方式有四种：1、CSV数据文件设置（CSV Data Set Config）：这种方式是通常所指的参数化。数据存储在文件中，该种参数化方式取值范围大，灵活性强，适用于大量测试数据时的使用。

1.1 用户参数（User Parameter）

​ 适用于参数取值范围很小，需要少量测试数据时使用。

1.2 用户自定义变量（User Defined Variables）

​ 更多用于设置全局变量，常用于数据库地址，测试环境、开发环境地址等常量配置。

1.3 函数助手

​ 可使用函数生成随机数字和随机字符串实现参数化。本文以前置处理器中的用户参数组件为例，实现JMeter中的参数化。

  • 添加变量：横着添加一行变量信息，此变量会应用于每个用户。
